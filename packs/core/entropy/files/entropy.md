# 복잡도/엔트로피 예산 (전 스택)

> 코드 엔트로피(복잡도·중복·크기) 증가를 능동적으로 억제한다. 한도 초과는 분해 신호.

## 예산 (기본 한도)
| 항목 | 한도 |
|------|------|
| 파일 | ≤ 300줄 |
| 함수/메서드 | ≤ 50줄 |
| 순환복잡도(cyclomatic) | ≤ 10 |
| 중첩 깊이 | ≤ 3 (초과 시 early return·추출) |
| 중복 | 3회 이상 반복 시 추출 (단 AHA — 과추상화 금지, `architecture.md`) |

## DO
1. 한도 초과 시 **분해** — 함수 추출·모듈 분리. 한 함수 = 한 책임.
2. early return으로 중첩 평탄화. 매직값은 명명 상수로.
3. 추가 전에 삭제 — 죽은 코드·미사용 export 제거.

## DON'T
1. "한 줄 더" 누적으로 거대 파일/함수 방치.
2. 한도 우회용 무의미 분할(가독성 해치는 과분할).

## 측정·정합
- 한도는 **기준선**(리뷰에서 근거 있으면 예외 가능, 기본은 거부). 하드 차단은 아님.
- Biome `complexity` 그룹 `noExcessiveCognitiveComplexity`로 부분 자동화 가능 — 단 **cognitive ≠ cyclomatic**(의미 다름). ccx 기본 `biome.json`은 complexity off이며, 프로젝트가 켜서 강제할 수 있다.

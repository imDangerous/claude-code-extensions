# 에이전트 작업 흐름

> AI 에이전트의 표준 작업 절차. 사람이 다시 지시하지 않아도 같은 품질로 재현되는 것이 목표.

## DO
1. **계획을 문서로** — 3파일 이상/다중 세션 작업은 `docs/plans/{날짜}-{slug}.md`에 먼저 저장.
2. **가정을 먼저 명시** — 해석이 갈리면 구현 전 확인.
3. **작업 단위 = 커밋 단위** — "그 김에 고친" 코드는 별도 커밋.
4. **광범위 탐색은 서브에이전트에 위임** — 메인 컨텍스트는 단일 파일·문서·대화에 보존.
5. **검증 후 종료** — lint·typecheck·test 통과해야 "완료".

## DON'T
1. 문서 갱신 없이 코드만 쌓지 않는다.
2. 로컬 검증 없이 push·PR 하지 않는다(`--no-verify` 금지).
3. 계획 없이 대규모 리팩터 시작하지 않는다(5파일↑ 사전 확인).
4. "작동한다"를 실행 없이 선언하지 않는다.

## 에스컬레이션
변경이 계획의 2배↑ / 기존 테스트 3개↑ 수정 / 외부 API 불확실 / 보안·권한·개인정보 / 동일 증상 2회↑ 실패 → 자동 진행 말고 보고.

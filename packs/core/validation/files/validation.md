# 입력 검증 (원칙 — 전 스택)

> 모든 신뢰 경계에서 외부 입력을 검증한다. 검증되지 않은 데이터는 신뢰하지 않는다(fail-closed).

## DO
1. **경계에서 검증** — 요청/폼/쿼리/헤더/env/파일/외부 API 응답 등 모든 외부 입력을 진입 경계에서 스키마로 검증·정규화.
2. **parse, don't validate** — 검증과 동시에 타입이 좁혀진 값을 만든다(검증 통과 = 타입 보장).
3. **fail-closed** — 검증 실패는 명시적 거부(기본 거부). 누락/모호를 통과시키지 않는다.
4. **신뢰 경계 명시** — 무엇이 신뢰/비신뢰 입력인지 문서화. 서버 권한·인가는 경계마다 **독립** 확인.
5. 출력도 경계에서 인코딩/이스케이프(XSS·인젝션 방지). 스키마는 한 곳을 SoT로.

## DON'T
1. 클라이언트 검증만 신뢰 — 서버에서 재검증 필수.
2. 부분 검증·검증 우회 경로·조용한 강제변환(silent coercion).
3. 검증 로직 산재(중복 스키마).

## 스택 구현
- JS/TS: zod (`validation-zod` — RN/web 공유).
- Spring: Bean Validation(`@Valid`/JSR-380) (`validation-backend`).

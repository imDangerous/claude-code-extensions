# web-ideate — 웹 제품 아이디어 리서치

웹/SaaS 아이디어를 리서치해 후보를 제안한다. 실행은 `web-idea-researcher` 에이전트에 위임.

## Trigger
"아이디어 줘", "뭐 만들까", "시장/경쟁 조사", "어떤 서비스 만들면 좋을까".

## Steps
1. **방향 설정** — 관심 도메인, 타겟(B2B/B2C·시장), 규모(1인/소규모).
2. **트렌드/수요 리서치** — WebSearch로 웹/SaaS 트렌드 + SEO 검색 수요·페인포인트.
3. **경쟁 서비스 분석** — 상위 서비스 5~10개 기능·가격·강약점, 차별화 기회.
4. **아이디어 도출** — 3~5개: 이름(가안)·한줄 설명·타겟·핵심 기능 3·차별점·수익 모델·SEO 기회·Next.js 실현 가능성.
5. **선택·저장** — 후보 제시 → 선택/조합 → `_workspace/01_idea_research.md`.

## 위임
전 단계 `web-idea-researcher`. 다음: `plan-web`(기획/PRD).

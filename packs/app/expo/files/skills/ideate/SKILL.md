# ideate — 앱 아이디어 리서치

앱 아이디어를 체계적으로 리서치해 실현 가능한 후보를 제안한다. 실행은 `idea-researcher` 에이전트에 위임.

## Trigger
"아이디어 줘", "앱 뭐 만들까", "어떤 앱이 좋을까", "트렌드/시장 조사".

## Steps
1. **방향 설정** — 관심 카테고리(없으면 전체 트렌드), 타겟 시장(한국/글로벌), 개발 규모(1인/소규모) 확인.
2. **트렌드 리서치** — WebSearch로 App Store/Play 인기·신규 트렌드, 페인포인트 많은 영역 탐색.
3. **경쟁 앱 분석** — 상위 앱 5~10개 핵심 기능·강약점, 리뷰의 반복 불만 → 차별화 기회.
4. **아이디어 도출** — 3~5개: 이름(가안)·한줄 설명·타겟·핵심 기능 3·차별점·수익 모델·Expo 실현 가능성.
5. **선택·저장** — 후보 제시 → 사용자 선택/조합 → `_workspace/01_idea_research.md`에 저장.

## 위임
전 단계 `idea-researcher`. 다음 단계는 `plan-app`(기획/PRD).
